# Code of Conduct

## Purpose

This code of conduct outlines expectations for participation in the Kernels community and the consequences for unacceptable behavior.

## Expected Behavior

All participants are expected to:

1. **Be respectful.** Treat all participants with respect regardless of background, identity, or experience level.

2. **Be constructive.** Provide feedback that is actionable and focused on improving the project.

3. **Be professional.** Communicate in a manner appropriate for a technical project.

4. **Be collaborative.** Work together toward shared goals.

5. **Be accountable.** Accept responsibility for mistakes and work to correct them.

## Unacceptable Behavior

The following behaviors are not tolerated:

1. Harassment, discrimination, or intimidation in any form.

2. Personal attacks or inflammatory comments.

3. Publishing private information without consent.

4. Disruptive behavior that interferes with productive discussion.

5. Advocating for or encouraging any of the above.

## Scope

This code of conduct applies to all project spaces including:

| Space                  | Coverage                           |
|------------------------|------------------------------------|
| Repository             | Issues, pull requests, discussions |
| Communication channels | Any official project channels      |
| Events                 | Any project-related gatherings     |
| Representation         | Acting as project representative   |

## Enforcement

Violations of this code of conduct may result in:

1. Warning from maintainers.

2. Temporary ban from project spaces.

3. Permanent ban from project spaces.

Maintainers have the authority and responsibility to remove, edit, or reject contributions that violate this code of conduct.

## Reporting

Report violations to the project maintainers. All reports will be reviewed and investigated. Reporters will be informed of the outcome.
